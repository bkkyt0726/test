# CLAUDE.md

## 프로젝트 개요
이 프로젝트는 사용자용 웹 서비스다.

기본 스택:
- 프론트엔드: Next.js + TypeScript
- 백엔드 API: Python + FastAPI
- 배포 기준: Vercel 친화적 구조 우선

기본 목표:
- 가장 작은 범위의 안전한 변경을 한다
- 기존 구조와 관례를 최대한 유지한다
- 요청 범위를 벗어난 수정은 하지 않는다

## 작업 원칙
- 수정 전에 관련 파일과 흐름을 먼저 확인한다
- 기존 코드 스타일과 폴더 구조를 우선 따른다
- 변경 범위는 최대한 작게 유지한다
- 관련 없는 파일은 수정하지 않는다
- 꼭 필요하지 않으면 새 의존성을 추가하지 않는다
- 실제 실행하지 않은 결과를 통과했다고 말하지 않는다

## 역할 분리 원칙
- UI, 화면 상태, 사용자 인터랙션은 프론트에서 처리한다
- 비즈니스 로직, 검증, 인증, 외부 연동, 민감한 처리는 백엔드에서 처리한다
- 프론트에는 민감한 로직이나 비밀값을 두지 않는다
- API 계약은 임의로 깨지 않는다

## 프론트엔드 규칙
- Next.js의 기존 구조(App Router / Pages Router)를 유지한다
- 기존 공용 컴포넌트와 디자인 패턴을 우선 재사용한다
- UI 컴포넌트에 비즈니스 로직을 과도하게 넣지 않는다
- 불필요한 `use client` 사용을 피한다
- 환경변수는 공개용과 서버 전용을 구분한다
- 요청 범위를 벗어난 디자인 변경은 하지 않는다

## 백엔드 규칙
- FastAPI 라우트 핸들러는 얇게 유지한다
- 비즈니스 로직은 서비스 함수 또는 별도 계층으로 분리한다
- 입력값은 반드시 검증한다
- 응답 형식과 상태 코드는 기존 계약을 우선 유지한다
- 인증, 권한, 보안 처리는 명시적으로 수행한다
- 민감한 정보는 로그나 응답에 노출하지 않는다

## 연동 규칙
- API 응답 필드, 타입, null 가능성 변경 시 프론트 영향 범위를 함께 확인한다
- 프론트 임시 보정보다 백엔드 계약 정합성을 먼저 확인한다
- 에러 응답은 프론트에서 처리 가능하도록 일관되게 유지한다

## 타입 / 스키마 규칙
- TypeScript와 Python 모두 새 코드에는 명확한 타입을 작성한다
- `any`, `Any` 사용은 최소화한다
- 데이터 구조가 바뀌면 타입과 스키마도 함께 수정한다

## 보안 / 배포 규칙
- 비밀키, 토큰, 인증정보를 하드코딩하지 않는다
- Vercel 환경 제약을 고려해 로컬 전용 가정을 피한다
- 서버리스 환경에서 긴 작업, 무거운 처리, 지속 연결은 주의한다

## 테스트 / 검증 규칙
변경 후 가능한 범위에서 아래를 우선 검증한다.

프론트엔드:
```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

백엔드:
```bash
ruff check .
mypy .
pytest
```

## 문서 / 변경 관리 규칙
- 실행 방법, 환경변수, API 계약이 바뀌면 문서도 함께 수정한다
- diff는 작고 리뷰하기 쉽게 유지한다
- 명시적으로 요청받지 않았다면 commit 하지 않는다

## 금지사항
- 요청 없는 대규모 리팩토링 금지
- 관련 없는 파일 수정 금지
- 프론트에 민감 로직 배치 금지
- 검증하지 않았는데 완료된 것처럼 말하지 말 것
- 실행하지 않은 테스트 결과를 지어내지 말 것
